# CB247 Marketing OS

AI-powered marketing automation for **ChasingBetter247 Health & Fitness Club**
(Malaga + Ellenbrook, Perth WA · 8,000+ members · $11.95/week).

Runs every **Monday 10:00 AWST** via cron. Pulls live data from 8 platforms,
emits actionable recommendations through a mix of rule-based emitters + LLM
strategists, syncs to a Supabase work queue, and presents everything on a
realtime dashboard at https://cb247agent.github.io/cb_claude/.

## Where do I…?

| Need to | Look at |
|---|---|
| **Run a Monday pipeline manually** | `bash scripts/weekly-report.sh` (logs to `state/weekly-report.log`) |
| **Check what's deployed** | https://cb247agent.github.io/cb_claude/ — the live dashboard |
| **Run pre-push QA checks** | `bash scripts/qa.sh` (Wave A + C combined, ~2 min with visual reg) |
| **Add a new emitter** | `scripts/work_queue/` — `ENGINEERING.md` has the 9-step cookbook |
| **Add a new LLM agent** | `agents/*.yml` — see `seo-strategist.yml` for the modern pattern + `agents/AGENT_ACTION_CONTRACT.md` for the output contract |
| **Find an action's brief** | `docs/briefs/{action-id}.html` — auto-regenerated by Phase 1 |
| **Update brand voice rules** | `scripts/work_queue/compliance.py` (banned-phrase patterns) + `CB_Brain/wiki/CB247-Knowledge-Base.md` (DOs/DON'Ts narrative) |
| **Refresh dashboard baselines after UI change** | `.venv/bin/python3.13 scripts/visual_regression.py --update-baselines` then commit `docs/baselines/` |
| **Run the SQL migrations** | Open Supabase Dashboard → SQL Editor → paste contents of `db/migrations/*.sql` |
| **See what failed last Monday** | `state/weekly-report.log` (timestamped) + `logs/run-YYYYMMDD-*.json` (structured) |
| **Switch brand context (CB247 ↔ MWCC)** | `python scripts/set_active_business.py mwcc` (or `cb247`) |

## The three-layer dev cycle (catches bugs before you see them)

Built in **Waves A + B + C (11–12 Jun 2026)** after hitting too many issues in production:

| Layer | Lives in | Catches |
|---|---|---|
| **A — Deterministic scripts** | `scripts/dev-cycle.sh --pre-commit` | Stale workflow vocab · Schema drift (Python ↔ SQL ↔ Supabase) · Strategist chain regressions · Vulnerable deps |
| **B — LLM audit agents** | `agents/qa-agent.yml` + `agents/security-agent.yml` (wired into Monday cron) | Wrong row count · Wrong list classification · Stale rule-emitter row leak · Leaked secrets · RLS regression |
| **C — Visual regression** | `scripts/visual_regression.py` + `docs/baselines/` | Wrong column rendering · Broken modal spacing · Accent color drift · Layout reflow |

One-shot pre-push command: `bash scripts/qa.sh` (runs A + C, skips B because B costs Claude tokens).

Full documentation: `HANDOFF.md` sections 8.5 + 8.6.

## Tech stack (the short version)

- **Python 3.13** for data pulls, emitters, and most scripting
- **Bash** for orchestration (weekly-report.sh + dev-cycle.sh + qa.sh + phases)
- **Supabase** for the realtime work queue (`work_queue_actions`, `planner_status`, `planner_approval`)
- **GitHub Pages** for hosting the dashboard (deploys on every push to `docs/`)
- **Claude Code** (Claude Max subscription) for all LLM agents
- **Playwright + Pillow** for visual regression
- **PostgreSQL CHECK constraints + RLS** for data integrity

No build system — content is generated, not compiled. State files in `state/*.json` are gitignored
(live data). Outputs in `outputs/` are date-stamped artifacts of each Monday run.
